# L1 - High-Level Requirements

## Cooperative Management Platform

This document defines the high-level requirements for the Coop platform, a cooperative residential management system delivered as a modular monolith with two application surfaces: a CMS-driven public-facing web app and an authenticated admin backend.

---

### L1-REQ-001: User Account Management

The system shall provide user account creation, authentication, and lifecycle management including secure credential storage and soft-delete capabilities.

---

### L1-REQ-002: Authentication and Authorization

The system shall authenticate users via username/password credentials, issue JWT tokens, and enforce role-based access control (RBAC) with resource-level privilege granularity.

---

### L1-REQ-003: Role and Privilege Management

The system shall support configurable roles (Member, Staff, BoardMember, SystemAdministrator, Support) with associated privileges that control access rights (Read, Write, Create, Delete) per resource aggregate.

---

### L1-REQ-004: Profile Management

The system shall support multiple profile types (Member, BoardMember, StaffMember, OnCall) per user, allowing users to operate under different contexts with a current and default profile selection.

---

### L1-REQ-005: Maintenance Request Workflow

The system shall manage maintenance requests through a defined lifecycle (New, Received, Started, Completed) with status tracking, comments, photo evidence, and audit history via event sourcing.

---

### L1-REQ-006: Document Management

The system shall support creation, editing, publishing, and deletion of cooperative documents including Notices, ByLaws, and Reports with optional file attachments.

---

### L1-REQ-007: Messaging System

The system shall provide a conversation-based messaging capability between profiles, supporting message creation, read tracking, and conversation management.

---

### L1-REQ-008: Digital Asset Management

The system shall support upload, retrieval, and deletion of digital assets (images, files) with metadata tracking (dimensions, content type) for use as avatars, document attachments, and maintenance request evidence.

---

### L1-REQ-009: Invitation and Onboarding

The system shall support invitation-based onboarding via tokens that are type-specific, optionally time-limited, and single-use, enabling new members to join the cooperative.

---

### L1-REQ-010: Theme and Content Customization

The system shall allow customization of visual themes (CSS custom properties) at global or per-user levels, and management of CMS-driven content sections used by the public-facing web app.

---

### L1-REQ-011: Event Sourcing and Audit Trail

The system shall persist domain events as an immutable audit log, capturing all state changes to aggregate roots with full event replay capability.

---

### L1-REQ-012: Modular Monolith Architecture

The system shall be implemented and deployed as a modular monolith with clear module boundaries for Identity, Profile, Maintenance, Document, Asset, Messaging, and CMS capabilities, sharing one codebase and one deployable backend.

---

### L1-REQ-013: Application Surfaces and API Layer

The system shall provide two applications, a CMS-driven public-facing web app and an admin backend, both backed by a shared RESTful HTTP API with appropriate HTTP status codes, pagination support, and authorization enforcement on protected endpoints.
